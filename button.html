Button

Кнопка

Использование

Если в макете явно выделены типы кнопок (семантически или визуально), то лучше использовать модификатор Внешнего вида (view-тип-кнопки), группирующий свойства фона, текста, границы и проч. Если же чёткой логики в оформлении кнопок нет, то лучше использовать модификаторы по названию свойств, которые они изменяют: отдельно для цвета фона, отдельно для цвета текста и тд

Заблокированные и находящиеся в активном состоянии интерактивные элементы не должны попадать в фокус при навигации с клавиатуры. Для этого кнопкам с тегом <a> добавляется атрибут tab-index="-1", с тегом <button> -- атрибут disable. Если не нужно поддерживать Safari 15, вместо tab-index="-1" можно использовать атрибут inert

Компоненту задаются стили по умолчанию и затем по необходимости переопределяются

Не стоит использовать свойство opacity без крайней необходимости, т.к. это может исказить визуал (при использовании кнопки поверх цветных элементов) и ухудшить читаемость при смене цветовой темы. Вместо opacity по возможности лучше использовать альфа-канал

В компоненте предусмотрена div-обёртка для возможности задания position: relative без влияния на позиционирование кнопки относительно других элементов на странице

Примеры

Заблокированная кнопка

Добавляет класс .is-disabled и атрибут tabindex="-1" или disabled

Показать код 
+button({
    isDisabled: true,
    text: "Button"
  })

Кнопка в активном состоянии

Добавляет класс .is-active и атрибут tabindex="-1" или disabled

Показать код 
+button({
    isActive: true,
    text: "Button"
  })

Кнопка с иконкой справа

Показать код 
+button({
    text: "Button",
    icon: {
      name: 'icon-arrow',
      width: 10,
      height: 16,
    },
  })

Кнопка с иконкой слева

Показать код 
+button({
    text: "Button",
    icon: {
      name: 'icon-arrow',
      width: 10,
      height: 16,
      isLeft: true,
    },
  })

Базовый компонент

Показать шаблон 
mixin button(props = {})
  -
    const
          tag = props.tag ? props.tag : props.href ? "a" : "button",
          href = props.href,
          text = props.text || "",
          type = props.type || tag === "button" && "button",
          target = props.isExternal && "_blank",
          rel = props.isExternal && "nofollow noopener noreferrer",
          icon = props.icon,
          {name, width, height, isLeft} = icon || {},
          isActive = props.isActive,
          activeClass = isActive && "is-active",
          isDisabled = props.isDisabled,
          disabledClass = isDisabled && "is-disabled",
          disabledAttr = (isDisabled || isActive) && tag === "button" && "true",
          tabindexAttr = props.tabindex || ((isDisabled || isActive) && tag === "a" && "-1"),
          modifiers = props.modifiers ? props.modifiers.map((mod) => "button--" + mod) : [];

  div(class=["button", disabledClass, activeClass, ...modifiers])&attributes(attributes)
    #{tag}.button__control(href=href, target=target, rel=rel, type=type, disabled=disabledAttr, tabindex=tabindexAttr)

      if (icon && isLeft)
        +icon(name, width, height)

      span.button__text !{text}

      if (icon && !isLeft)
        +icon(name, width, height)

Внешний вид (комбинирование декоративных модификаторов)

Типизация может быть только визуальной, необязательно семантической

Базовый

Без модификатора

Показать код 
+button({
  text: "Button"
})
Clear

Просто кнопка с прозрачным фоном

Показать код 
+button({
  modifiers: ["view-clear"],
  text: "Button"
})
Primary

Основная кнопка для важных действий, к которым нужно привлечь внимание. Также подходит для случаев, когда в проекте используется один вариант кнопки

Показать код 
+button({
  modifiers: ["view-primary"],
  text: "Button"
})
Secondary

Подходит для важных действий, к которым не нужно привлекать особое внимание

Показать код 
+button({
  modifiers: ["view-secondary"],
  text: "Button"
})
Ghost

Градиент при наведении

Для создания эффекта плавной анимации фона с градиентом кнопке добавляется псевдоэдемент с нужным градиентом и скрывается через opacity: 0. При наведении на кнопку opacity псевдоэлемента возвращается в 1

Не стоит в состоянии hover задавать прозрачность основному фону кнопки, т.к. кнопка начнёт "мигать"

p.s. когда поддержка правила @property станет получше, анимировать градиент будет, если не проще, но точно интересней :)

Показать код 
+button({
  modifiers: ["active-gradient", "filled-brand", "color-base-essential"],
  text: "Button"
})

&--active-gradient {
  .button__control {
    position: relative;

    &::before {
      position: absolute;
      top: calc(var(--border-width) * -1);
      right: calc(var(--border-width) * -1);
      bottom: calc(var(--border-width) * -1);
      left: calc(var(--border-width) * -1);
      content: "";

      background-image: $gradient-linear-45-project-to-essential;
      border-radius: var(--border-radius);

      opacity: 0;
      transition: opacity $trans-default;
    }
  }

  .button__text,
  svg {
    z-index: 1;
  }

  @include interactive-style {
    .button__control::before {
      opacity: 1;
    }
  }
}
